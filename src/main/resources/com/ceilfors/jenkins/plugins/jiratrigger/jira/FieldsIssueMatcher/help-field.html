The JIRA Field name (both builtin such as “Issue Type” and customfields).

All the field names for your Jira server is provided, you need to select one (if it's not displayed, you need type a few letters first for it to be filtered).

Note: This is intended to replace the old jqlFilter setting. jqlFilter won't work for Jira Data Center version, because of the following scenario:

  1. User create a Jira Issue on Node A.
  2. JTP (Jira Trigger Plugin, this one) searches jql on Node B.

This won't work, because for jql search to work, the Jira search index need to be in SYNC, but Jira does not assure it's realtime.
